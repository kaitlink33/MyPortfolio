Answer 1.)
I picked num = 108 for this test case. Its prime factorization is 1*2*2*3*3*3.
The most frequent factor, 3, occurs 3 times.
It passes input validation because num <= 0 is not true.
All of my initial variables are the same, besides num. 
I print the start of my factorization statement, and then add factors 
as the program finds them.
This process begins at 2 and checks every number up to num for divisibility. 
If divisible, the number is added to the statement however many times it is divisible and a count is kept
accordingly inside currentDivisorTimes, which is compared against highestDivisorTimes for each iteration. 
If greater (or equal), the current divisior var is changed to the mostCommonDivisor with preference to the 
larger number. The value of currentDivisorTimes is reset each time divisor increases. In this case, mostCommonDivisor = 3
and highestDivisorTimes = 3.
